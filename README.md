## Опис завдань

### Завдання 1: Метод delete для HashTable
Додано метод `delete` для видалення пар ключ-значення з хеш-таблиці. Метод коректно обробляє випадки видалення існуючих та неіснуючих ключів.

### Завдання 2: Двійковий пошук з верхньою межею
Реалізовано двійковий пошук для відсортованого масиву дробових чисел. Функція повертає кортеж `(кількість_ітерацій, верхня_межа)`, де верхня межа - це найменший елемент, який є більшим або рівним заданому значенню.

### Завдання 3: Порівняння алгоритмів пошуку підрядка
Проведено порівняльний аналіз ефективності трьох алгоритмів пошуку підрядка: Боєра-Мура, Кнута-Морріса-Пратта та Рабіна-Карпа на двох текстових файлах.

### Загальні висновки

1. **Бойер-Мур** показав найкращі результати для обох текстів та обох типів підрядків (існуючих та вигаданих). Алгоритм ефективно використовує евристику "поганого символу" для швидкого пропуску непотрібних порівнянь.

2. **KMP** займає друге місце. Алгоритм ефективний для текстів з повторюваними паттернами завдяки використанню таблиці LPS (Longest Proper Prefix which is also Suffix), але все ж поступається Бойер-Муру для даних тестових випадків.

3. **Рабін-Карп** показав найгірші результати. Хоча алгоритм має теоретичну складність O(n+m), на практиці він виявився значно повільнішим через накладні витрати на обчислення хешів та обробку колізій.

4. **Вплив типу підрядка**: Для вигаданих підрядків (які не існують у тексті) час виконання збільшується для всіх алгоритмів, оскільки вони мають перевірити весь текст перед поверненням негативного результату.

5. **Вплив розміру тексту**: Стаття 2 має більший розмір, що призводить до збільшення часу виконання для всіх алгоритмів, але відносне співвідношення швидкості залишається незмінним.
